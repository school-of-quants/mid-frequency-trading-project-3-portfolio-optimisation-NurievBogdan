## Важные замечания по запуску

Полный запуск на исторической вселенной S&P 500 может быть медленным, потому что Yahoo Finance не всегда возвращает данные по всем историческим и исключённым из индекса тикерам. Код сохраняет файл `missing_tickers.json`, чтобы отсутствующие активы можно было диагностировать или вручную подгрузить из другого источника данных.

Для отладки установите в `config.yaml`:

```yaml
debug: true
debug_n_tickers: 80
```

После отладки переключите параметр обратно на `debug: false` для финального запуска.

## Краткое описание стратегии

Стратегия представляет собой **long-only mid-frequency equity-портфель** на исторической вселенной S&P 500. Основная идея — ранжировать акции внутри текущей индексной вселенной по вероятности относительного опережения рынка и еженедельно держать лучшие бумаги с ограничениями по весам и риску.

### Данные и вселенная

- Используются дневные OHLCV-данные акций, входивших в S&P 500, с учётом исторического состава индекса, чтобы уменьшить survivorship bias.
- Период обучения и валидации: **2017-01-01 — 2022-12-31**.
- Период итогового бэктеста: **2023-01-01 — 2025-12-31**.
- Бенчмарк для сравнения — **SPY / S&P 500**.

### Фичи

- Momentum/return-факторы на окнах **1, 5, 10, 21, 63, 126 и 252** торговых дня.
- Cross-sectional ranks и z-score momentum, чтобы модель учила относительную силу бумаг, а не только общий рыночный тренд.
- Отклонения от moving averages, включая **50/200 MA**.
- Волатильность и ранги волатильности на нескольких окнах.
- Intraday/overnight returns, high-low range, volume z-score, dollar-volume rank и Donchian position.
- Все признаки сдвинуты на один торговый день, чтобы не использовать информацию из будущего.

### Target и модель

- Target — трёхклассовая метка по **10-дневной будущей избыточной доходности** акции относительно среднего по дневной вселенной.
- Класс `2` соответствует верхним 30% акций по forward excess return, класс `0` — нижним 30%, класс `1` — нейтральной зоне.
- Для прогнозирования используется `CatBoostClassifier` с `MultiClass` loss и балансировкой классов.
- Качество модели проверяется через `PurgedKFold` с purge-периодом **10 дней** и embargo **1%**, а также через `Combinatorial Purged Cross-Validation`.

### Построение портфеля

- Для каждой акции считается скор: `P(outperform) - P(underperform)`.
- Ребалансировка проводится еженедельно, на последнюю доступную торговую дату недели.
- В портфель попадают до **50 акций** с положительным скором и максимальным рангом.
- Веса пропорциональны положительным скорам, но ограничены сверху **3%** на одну бумагу.
- Максимальная gross exposure ограничена **95%**, поэтому стратегия не использует плечо и оставляет часть капитала в кэше.
- Исполнение моделируется как TWAP-прокси: цена сделки равна среднему значению `Open`, `High`, `Low` и `Close` за день.
- Комиссия учитывается как **0.1%** на сделку.

### Оценка

- Итоговый бэктест считает total return, Sharpe ratio, Calmar ratio, maximum drawdown, длительность drawdown и проверку ограничений.
- Дополнительно рассчитывается intra-portfolio correlation (**IPC**) портфеля и сравнивается с IPC вселенной S&P 500.
- В архиве нет готовых сохранённых метрик бэктеста, поэтому фактическое превосходство над S&P 500 по Sharpe ratio нужно подтверждать запуском полного пайплайна.
